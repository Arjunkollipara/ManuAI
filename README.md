# Manufacturing Quality & Productivity Suite

AI-powered capstone platform for predictive maintenance, manufacturing analytics, RAG search, and agent-assisted operations.

## What This Project Does

- FastAPI backend for ingestion, prediction, search, agents, reporting, and health checks
- ML pipeline for training and inference
- Cyber-industrial frontend served at `/ui`
- SQLite runtime database with Power BI export support
- Azure integration hooks for OpenAI, Azure AI Search, and Azure SQL configuration
- Docker and Azure App Service deployment support

## Live Routes

- `GET /health` - runtime health check
- `GET /ui/` - frontend dashboard
- `GET /system/config` - environment readiness snapshot
- `POST /ingest/sensor-data` - ingest sensor readings
- `POST /predict/failure` - run a prediction
- `GET /search/documents` - RAG document search
- `GET /search/documents/health` - RAG health
- `GET /reports/summary` - reporting summary
- `GET /reports/power-bi` - Power BI export JSON or CSV

## Actual Runtime Database

The application currently runs on SQLite.

- Runtime URL: `sqlite:///./manufacturing.db` in local `.env`
- Docker URL: `sqlite:///./data/manufacturing.db`
- SQLAlchemy engine: `api/database/connection.py`

Azure SQL credentials may exist in environment variables, but they are not the active runtime database unless `DATABASE_URL` is changed and the app is redeployed against Azure SQL.

## Requirements

- Python 3.11
- `pip`
- Optional: Docker Desktop
- Optional: Azure App Service / Azure SQL / Azure AI Search / Azure OpenAI

## Local Run

### 1. Create or activate the virtual environment

On Windows `cmd`:

```bat
env311\Scripts\activate
```

If you are using a different environment:

```bat
env\Scripts\activate
```

### 2. Install dependencies

```bat
pip install -r requirements.txt
```

### 3. Start the API

```bat
uvicorn api.main:app --reload --host 127.0.0.1 --port 8000
```

### 4. Open the UI

```text
http://127.0.0.1:8000/ui/
```

## Docker Run

Build and run the container:

```bat
docker build -t manufacturing-suite .
docker run --rm -p 8000:8000 manufacturing-suite
```

Open:

```text
http://127.0.0.1:8000/ui/
```

## Docker Compose

```bat
docker compose up --build
```

The compose service maps `PORT` if present and persists:

- `./data` to `/app/data`
- `./logs` to `/app/logs`
- `./ml/models` to `/app/ml/models`
- `./documents` to `/app/documents`
- `./reports` to `/app/reports`

## Azure App Service Startup

Use this startup command:

```text
uvicorn api.main:app --host 0.0.0.0 --port ${PORT:-8000} --workers 1 --proxy-headers --forwarded-allow-ips='*' --access-log --log-level info
```

The container and App Service path both bind to `0.0.0.0` and honor `PORT`.

## Health Checks

- `GET /health` returns `healthy` when startup completed and the database ping succeeds
- If the app is not ready, `/health` returns `503` with `degraded`

## Power BI Setup

Use one of these:

1. `GET /reports/power-bi` with `format=json`
2. `GET /reports/power-bi?format=csv`
3. The SQLite database file generated by the live app

Recommended for the current architecture:

- Connect Power BI to the CSV export or SQLite-backed reporting data
- Do not point Power BI at Azure SQL unless you migrate the runtime database to Azure SQL first

## Deployment Notes

- Static UI is served from `frontend/`
- `/ui` is the live frontend route
- `rebuild_html.py` no longer overwrites the live UI
- Optional Azure services are allowed to be absent at boot
- SQLite is the active runtime database until explicitly migrated

## Troubleshooting

- If `/ui` is blank, restart the backend and hard refresh the browser
- If `/health` returns `503`, check database access and startup logs
- If search returns empty results, verify documents exist in `documents/`
- If predictions fail, verify `ml/models/random_forest.pkl` and `data/curated/feature_metadata.json`
